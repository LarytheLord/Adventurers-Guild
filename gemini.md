This project uses Supabase to store waitlist data. The data is stored in a table named `waitlist` with the following schema:

```sql
CREATE TABLE waitlist (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email text NOT NULL UNIQUE,
  created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

The project has an API route at `app/api/waitlist/route.ts` that handles `GET` and `POST` requests to the `waitlist` table.

A `WaitlistForm` component at `components/waitlist-form.tsx` provides a form for users to join the waitlist. This form is displayed in a dialog on the main page.
